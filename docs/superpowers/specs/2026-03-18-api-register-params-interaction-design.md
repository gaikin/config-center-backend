# API注册 参数示例交互设计（契约定义版）

- 日期：2026-03-18
- 范围：`src/pages/InterfacesPage/*`
- 目标：将 `API注册 > 参数示例` 收敛为“接口契约定义层”，把值来源映射职责完全迁移到“智能提示”。

## 1. 背景与问题

当前“参数示例”同时承担了结构定义和运行时映射，造成以下问题：

1. 认知冲突：用户不知道应在 API 页还是智能提示页配置映射。
2. 结构歧义：`Body JSON` 与参数表都可改结构，容易产生不一致。
3. 复杂度过高：业务用户在定义阶段被迫处理映射细节，学习成本高。

## 2. 设计目标

1. API页只负责“契约定义”：字段结构、类型、描述、简单字段校验。
2. 智能提示页负责“运行时映射”：来源、值映射、业务逻辑。
3. Body 采用单一真实来源：`Body JSON`。
4. 交互可解释：用户能快速理解“该在何处做什么”。

## 3. 方案对比与结论

### 方案A（采用）

- API页：契约定义 + 字段级校验（必填、长度/范围、正则）
- 智能提示页：值来源绑定与业务条件
- Body：解析即全量重建结构

优点：边界稳定、心智清晰、便于后续扩展。

### 方案B（未采用）

- API页保留隐藏映射入口

缺点：职责再次混杂，容易回到旧问题。

### 方案C（未采用）

- 校验也全部放到智能提示

缺点：接口契约约束过弱，不利于复用和治理。

## 4. 信息架构与交互

### 4.1 参数示例职责边界

`API注册 > 参数示例` 保留：

1. 字段名
2. 路径（Body/出参）
3. 类型
4. 说明
5. 字段级校验（新）

移除：

1. 来源
2. 值/映射

### 4.2 Body 交互

1. `Body JSON` 为唯一结构来源。
2. 点击“解析并重建结构”后全量覆盖 Body 结构表。
3. 保留固定提示：“结构调整请修改 Body JSON 后重新解析”。
4. 取消“手动新增 Body 参数”入口。

### 4.3 字段级校验（第一版）

支持项：

1. 必填
2. 长度/范围
3. 正则（模板 + 自定义）

正则模式：

1. 常用模板（如手机号、邮箱、身份证）
2. 自定义正则表达式

展示策略：

1. 表格显示“校验摘要”列
2. 详细配置在“校验”弹层中完成，避免表格拥挤

## 5. 数据模型（新系统）

### 5.1 输入参数项

`inputConfigJson` 仅保留契约相关字段：

1. `name`
2. `description`
3. `valueType`
4. `validationConfig`（新增）

不再保存：

1. `sourceType`
2. `sourceValue`

### 5.2 校验配置结构

```ts
type FieldValidationConfig = {
  required?: boolean;
  string?: { minLen?: number; maxLen?: number };
  number?: { min?: number; max?: number };
  pattern?: {
    preset?: "MOBILE" | "EMAIL" | "ID_CARD" | "CUSTOM";
    expr?: string;
  };
};
```

## 6. 数据流

1. 用户编辑 `Body JSON`
2. 点击“解析并重建结构”
3. 前端解析为 Body 字段结构并覆盖当前 Body 结构
4. 用户为字段补充说明与校验配置
5. 保存时仅提交契约 + 校验
6. 智能提示页读取契约字段并做来源映射

## 7. 错误处理

1. Body JSON 为空：提示“请先输入 Body JSON”
2. Body JSON 非法：提示解析错误，保持当前结构不变
3. 校验配置非法（如 min > max、自定义正则无效）：阻止保存并就地提示

## 8. 测试与验收

### 8.1 关键用例

1. Body JSON 合法解析后，结构表与 JSON 一致
2. Body JSON 非法时，结构表不被污染
3. “来源/值映射”字段在 API 页不可见
4. 字段校验可保存并回显
5. 正则模板与自定义正则都可生效

### 8.2 验收标准

1. 业务用户能明确区分“契约定义”和“运行时映射”职责
2. API 页完成结构定义无需处理来源映射
3. 智能提示页可独立完成参数来源配置

## 9. 实施顺序

1. 收敛 `InterfacesPage` 参数示例 UI（移除映射输入）
2. 升级 `useInterfacesPageModel` 数据读写（去除映射字段）
3. 增加字段级校验配置 UI + 数据结构
4. 更新验证逻辑与保存逻辑
5. 联动验证智能提示侧读取契约并配置映射
